# 🕒 網路安全中的數字、日期與時間篩選

在日誌（Log）分析中，時間戳記是重建事件現場的關鍵。常見的資料類型包括：
* **登入日期/時間**：追蹤帳號活動。
* **補丁日期 (Patch Date)**：確認系統是否已更新漏洞修補。
* **連線持續時間**：辨識潛在的資料外洩（例如異常長時間的連線）。

---

## 1. 比較運算子 (Comparison Operators)
篩選數值、日期或時間時，會使用以下運算子：

| 運算子 | 說明 |
| :--- | :--- |
| `<` | 小於 |
| `>` | 大於 |
| `=` | 等於 |
| `<=` | 小於或等於 |
| `>=` | 大於或等於 |
| `<>` 或 `!=` | **不等於** |

### 整合至過濾器
這些運算子必須放在 `WHERE` 子句中：
```sql
SELECT *
FROM log_events
WHERE login_time > '2026-04-01';
